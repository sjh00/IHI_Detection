## 如何报告问题

如果您发现了一个bug或其他问题，请遵循以下步骤报告：

1. 搜索现有的GitHub Issues，确保该问题尚未被报告
2. 如果没有找到，请创建一个新的Issue
3. 提供清晰、简洁的问题描述
4. 包含重现问题的步骤
5. 提供您的环境信息（操作系统、Python版本、依赖版本等）
6. 如果可能，提供错误日志或屏幕截图

## 如何提出改进建议

如果您有改进项目的想法或建议，请遵循以下步骤：

1. 创建一个新的GitHub Issue
2. 清晰地描述您的想法或建议
3. 解释为什么这个改进对项目有好处
4. 如果可能，提供实现的思路或示例

## 如何提交代码

1. Fork项目到您的GitHub账户
2. 克隆您的Fork到本地
3. 创建一个新的分支，用于实现您的功能或修复
4. 编写代码，确保符合项目的代码风格
5. 运行测试，确保所有测试通过
6. 提交您的更改，并推送到您的GitHub账户
7. 创建一个Pull Request，将您的分支合并到主项目

## 开发流程

1. **功能开发**：在新的分支上开发新功能
2. **代码审查**：提交Pull Request后，其他贡献者将对您的代码进行审查
3. **测试**：确保所有测试通过
4. **合并**：审查通过后，代码将被合并到主分支
5. **发布**：定期发布新版本

## 代码风格

1. 遵循PEP 8 Python代码风格指南
2. 使用4个空格作为缩进
3. 变量和函数名使用蛇形命名法（snake_case）
4. 类名使用驼峰命名法（CamelCase）
5. 为复杂的代码添加注释
6. 保持代码简洁明了

## 测试要求

1. 为新功能编写单元测试
2. 确保所有测试通过
3. 避免任何测试失败的提交
4. 运行测试的命令：
   ```bash
   cd IHI_MCP
   python -m pytest tests/
   ```

## 文档要求

1. 为新功能更新文档
2. 保持README.md和其他文档的最新状态
3. 为复杂的功能提供详细的说明

## 许可证信息

- 所有贡献的代码将遵循GPLv3许可证
- 确保您有权贡献代码，并且代码不侵犯任何第三方版权

## 其他注意事项

1. 请尊重其他贡献者
2. 保持沟通友好和专业
3. 遵守项目的行为准则
4. 如果有任何疑问，请随时提问
